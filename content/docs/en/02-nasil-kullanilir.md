# How To Use It

Neta's usage flow starts by creating the first owner account. After that, the owner configures workspace settings, adds clients, creates projects, and starts using the client portal.

## First Sign In

After a fresh installation, the first thing to do is create the owner account.

Open this address in the browser:

```text
https://neta.example.com/register
```

When the first registration succeeds, the system accepts that account as the owner. Public registration is closed after that. This prevents another person from creating a second owner account from the same screen.

## Owner Account

The owner account is the main administrator of the system.

The owner can:

- Manage workspace settings.
- Choose logo, favicon, colors, and theme.
- Create clients.
- Create client portal accounts.
- Manage projects, tasks, finance, and journal records.
- Configure AI providers.
- Run backup and restore operations.

## First Settings

After the owner account is created, the recommended first step is opening the settings screen.

The owner enters:

- Workspace name
- Meta title
- Short application name
- Light logo
- Dark logo
- Favicon
- Primary color
- Theme preference

These values become the main brand data for the dashboard, client portal, and mobile discovery endpoints.

## Creating A Client

Clients are at the center of the workflow in Neta.

When creating a new client, the owner usually enters:

- Client name
- Company name
- Email
- Phone
- Notes
- Status

After the client is created, projects, finance records, and portal access can be attached to that client.

## Opening A Portal Account

To give a client portal access, the owner starts the invite or portal account flow from the client detail screen.

The general flow is:

1. The owner opens the client detail page.
2. The owner starts the portal account or invite creation process.
3. The system creates a one-time or time-limited invite link.
4. The client opens the link.
5. The client sets their own password.
6. The client account is attached to the related client record.

After signing in, the client can only see data that belongs to them.

## Creating A Project

Projects can be tracked with or without a client connection.

A project usually includes:

- Project name
- Related client
- Status
- Priority
- Start and delivery date
- Description
- Tasks
- Finance records

The project detail screen can be treated as the operations center for that piece of work.

## Task Tracking

Tasks are used to track the work that needs to be done inside a project.

Tasks can include:

- Title
- Description
- Status
- Priority
- Due date
- Related project

Tasks can be viewed as a list or as a kanban board. The kanban view is useful for quickly seeing which stage the work is in.

## Calendar Usage

The calendar manages the time-based side of the work.

It can contain:

- Meetings
- Delivery dates
- Client calls
- Project milestones
- Reminders

The calendar helps freelancers see not only what needs to be done, but also how that work is distributed across time.

## Creating Finance Records

The finance module is used for income and expense tracking.

Record types:

- Income
- Expense
- Pending payment

Finance records can be associated with clients or projects. This makes it easier to see how much income came from a client or how much cost belongs to a project.

## Journal Usage

The journal keeps the freelancer's work history.

Journal records can include work notes, mood, energy level, and short reflections. Over time, this area helps the owner understand their own work rhythm.

## AI Usage

AI features are optional.

The owner can configure one of the supported providers from settings. API keys are not sent to the browser; they are stored securely on the server side.

AI can be used for:

- Project risk analysis
- Finance interpretation
- General business analysis
- Chat
- Summaries and insights

## Example Daily Workflow

A typical Neta day can look like this:

1. Open the dashboard.
2. Check active projects and pending tasks.
3. Review the day's meetings in the calendar.
4. Add new clients or projects if needed.
5. Update completed work in tasks.
6. Enter income or expense records in finance.
7. Create a journal note at the end of the day.

## Working With Clients

The recommended client workflow is:

1. The owner creates the client record.
2. A project is created and attached to the client.
3. Required tasks are added.
4. The portal invite is sent to the client.
5. The client follows project status through the portal.
6. The owner updates the information visible in the portal when needed.

This structure reduces manual status messages with clients.

## Summary

The basic idea of using Neta is to prepare the system for your own brand first, then add clients and projects, and finally run daily operations through the dashboard, tasks, calendar, finance, and portal.
